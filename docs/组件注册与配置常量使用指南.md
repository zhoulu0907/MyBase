# Materials 组件注册与配置常量使用指南

## 适用范围
- 适用于在宿主或插件中注册组件、获取组件实现与配置，以及在组件 schema 中复用通用配置常量。
- 相关代码模块：`@onebase/ui-kit` 的 `Materials` 子模块。

## 快速上手
- 引入 API：
```ts
import {
  registerComponent,
  registerMaterialsPlugin,
  getComponentImpl,
  getComponentSchema,
  buildTemplate,
  buildDisplayNameMap,
  buildComponentFieldMap,
  buildEntityToComponentMap,
  getCommonConfig,
  COMMON_CONFIG,
  COMMON_CONFIG_GROUPS
} from '@onebase/ui-kit';
```

## 注册 API
- 单组件注册：在注册表中写入一个组件定义
```ts
registerComponent({
  type: 'XMyCustomInput',
  component: XMyCustomInput,
  schema: {
    editData: [
      COMMON_CONFIG.labelConfig,
      COMMON_CONFIG.placeholderConfig,
      COMMON_CONFIG.tooltipConfig,
      COMMON_CONFIG.verifyConfig,
      COMMON_CONFIG.statusConfig,
      COMMON_CONFIG.widthConfig
    ],
    config: {
      label: '',
      placeholder: '',
      tooltip: '',
      verify: { required: false },
      status: 'default',
      width: '100%',
      layout: 'vertical',
      cpName: '',
      id: ''
    }
  },
  template: { h: 36, w: 118, displayName: '自定义输入', icon: 'my_input_cp.svg', category: 'form' },
  fieldMap: ['TEXT'],
  entityMap: ['TEXT']
});
```
- 插件批量注册：一次性注册多个组件并记录插件与组件的关联
```ts
const myPlugin = {
  id: 'com.example.my-plugin',
  components: [ /* 若干组件描述对象 */ ]
};
registerMaterialsPlugin(myPlugin);
```
- 卸载与覆盖：
```ts
unregisterComponent('XMyCustomInput');
invalidateMaterialsPlugin('com.example.my-plugin');
registerComponent(descriptor, { override: true });
```

## 获取与渲染
- 获取组件实现与配置：
```tsx
const type = 'XMyCustomInput';
const Impl = getComponentImpl(type);
const schema = getComponentSchema(type);

return Impl ? <Impl {...schema.config} runtime detailMode={false} /> : null;
```

## 模板与映射（动态获取）
- 模板：`buildTemplate()` 获取物料面板展示数据（分类、名称、图标等）。
- 显示名称映射：`buildDisplayNameMap()` 获取 `type -> displayName` 映射。
- 组件→字段能力映射：`buildComponentFieldMap()` 获取表单组件可绑定的实体字段集合。
- 实体字段→默认组件映射：`buildEntityToComponentMap()` 用于自动选择组件。

说明：若插件在运行期才完成注册，务必使用上述“动态函数”获取最新数据；静态常量只在模块初始化时计算。

## 通用配置常量
- 便捷函数：
```ts
const width = getCommonConfig('widthConfig');
const status = getCommonConfig('statusConfig');
const editData = [width, status];
```
- 单键映射：`COMMON_CONFIG`（可直接取键值，如 `COMMON_CONFIG.widthConfig`）
- 分组映射：`COMMON_CONFIG_GROUPS`（按功能分组，便于批量组合）

## 最佳实践
- schema 的 `editData` 优先复用 `COMMON_CONFIG_GROUPS.text`、`layout`、`status` 分组；`config` 补齐对应默认值。
- 在应用初始化阶段尽早完成插件注册，以便静态常量（如 `FORM_COMPONENT_TYPES`、`ALL_COMPONENT_TYPES`）包含插件组件；否则使用动态函数获取最新数据。
- 组件实现接受并使用 `schema.config` 的键（如 `width`、`status`、`label`、`placeholder`、`tooltip`、`verify`、`dataField`、`runtime`、`detailMode`、`id`、`cpName`）。
