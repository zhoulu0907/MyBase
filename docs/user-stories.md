### FLOW-001: 设计并执行业务流程

**用户角色：** 业务流程设计师（张经理）

**需求描述：**
```
作为一个业务流程设计师，
我想要通过可视化界面设计业务流程并执行，
以便快速响应业务需求变化，无需等待IT开发。
```

**验收标准：**
- ✅ 能够在流程设计器中创建5个节点的流程
- ✅ 能够配置节点参数（数据映射、条件表达式）
- ✅ 能够连接节点定义执行顺序
- ✅ 流程执行响应时间 < 500ms
- ✅ 流程执行失败时返回明确错误信息
- ✅ 能够查看流程执行日志（每个节点的输入输出）

**优先级：** P0（必须有）

**故事点：** 8

**技术实现：**
- 核心类：`FlowProcessExecutor.java:44`
- Git提交：44次修改
- 关键文件：
  - `FlowProcessExecServiceImpl.java` (36次修改)
  - `FlowProcessExecApiImpl.java` (26次修改)

**依赖关系：**
- 依赖：FLOW-002（节点组件库）
- 依赖：FLOW-003（表达式引擎）
- 被依赖：FLOW-005（流程监控）

**业务价值：**
- 🎯 **效率提升**：从需求到上线仅需4小时（传统方式2周）
- 💰 **成本降低**：减少IT开发工作量
- ⚡ **响应速度**：流程调整即时生效

## 📈 故事点估算说明

### 估算基准

**故事点定义：**
- **1点**：简单任务，1-2小时完成
- **3点**：中等任务，1-2天完成
- **5点**：复杂任务，3-5天完成
- **8点**：非常复杂，1-2周完成
- **13点**：超复杂，2-4周完成

### 估算依据

**FLOW-001: 设计并执行业务流程（8点）**
- 流程执行器核心逻辑复杂
- 需要与LiteFlow引擎集成
- 需要处理异常和日志
- 需要性能优化（P95 < 500ms）
- **参考：** FlowProcessExecutor 44次修改

**FLOW-002: 使用节点组件库（5点）**
- 需要实现10+种节点
- 每个节点需要数据类和配置
- 需要统一异常处理
- 需要节点跳过逻辑
- **参考：** DataAddNodeComponent 22次修改

**FLOW-003: 使用表达式引擎（5点）**
- 需要集成MVEL和JEXL引擎
- 需要处理空值和异常
- 需要上下文变量访问
- 需要类型转换
- **参考：** ExpressionExecutor 16次修改

**ORM-002: 租户数据自动隔离（8点）**
- 需要拦截器机制
- 需要确保100%隔离
- 需要处理跨租户查询
- 需要安全验证
- **影响：** 所有数据访问

**APP-001: 管理应用菜单（5点）**
- 需要3级菜单树形结构
- 需要排序和权限控制
- 需要性能优化（LinkedList）
- 需要批量操作
- **参考：** AppMenuRepository 20次修改
