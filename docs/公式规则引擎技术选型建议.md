# OneBase 公式规则引擎技术选型与架构建议

## 技术选型推荐

### 表达式引擎：Aviator + MVEL
- **主引擎**：Aviator（高性能数学计算、复杂逻辑）
- **辅助引擎**：MVEL（动态模板、简单表达式）

### 规则引擎：Drools + Easy Rules
- **复杂规则**：Drools（企业级业务规则）
- **简单规则**：Easy Rules（轻量级快速规则）

## 架构设计建议

### 1. 模块结构设计
```
onebase-module-rule/
├── onebase-module-rule-api/           # 跨模块调用API
│   ├── dto/                          # 数据传输对象
│   └── api/                          # Feign接口定义
└── onebase-module-rule-server/        # 规则引擎核心服务
    ├── controller/                    # REST API控制器
    ├── service/                       # 业务逻辑层
    ├── repository/                    # 数据访问层
    ├── engine/                        # 引擎实现
    └── config/                        # 配置类
```

### 4. 配置类设计
```java
@Configuration
@EnableConfigurationProperties({AviatorProperties.class, DroolsProperties.class})
public class RuleEngineAutoConfiguration {
    
    @Bean
    @ConditionalOnMissingBean
    public AviatorEvaluatorInstance aviatorEngine() {
        AviatorEvaluatorInstance engine = AviatorEvaluator.newInstance();
        // 安全配置
        engine.setOption(Options.ALWAYS_PARSE_FLOATING_POINT_NUMBER_INTO_DECIMAL, true);
        engine.setOption(Options.ALWAYS_PARSE_INTEGRAL_NUMBER_INTO_DECIMAL, true);
        return engine;
    }
    
    @Bean
    @ConditionalOnMissingBean
    public KieContainer kieContainer() {
        KieServices kieServices = KieServices.Factory.get();
        return kieServices.getKieClasspathContainer();
    }
}
```

### 5. 安全考虑
- **沙箱模式**：启用Aviator沙箱，限制系统调用
- **权限控制**：Controller层使用@PreAuthorize控制访问权限
- **输入验证**：使用@Valid验证表达式和规则格式
- **资源限制**：设置表达式执行超时和内存限制

### 6. 性能优化
- **缓存机制**：缓存编译后的表达式和规则
- **连接池**：规则引擎会话池管理
- **异步执行**：复杂规则异步处理
- **监控告警**：集成Spring Boot Actuator监控

### 7. 数据库设计
```sql
-- 表达式模板表
CREATE TABLE rule_expression_template (
    id BIGINT PRIMARY KEY,
    name VARCHAR(100) NOT NULL COMMENT '表达式名称',
    expression TEXT NOT NULL COMMENT '表达式内容',
    engine_type VARCHAR(20) NOT NULL COMMENT '引擎类型：aviator,mvel',
    description TEXT COMMENT '描述',
    tenant_id BIGINT COMMENT '租户ID',
    created_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 业务规则表
CREATE TABLE rule_business_rule (
    id BIGINT PRIMARY KEY,
    rule_name VARCHAR(100) NOT NULL COMMENT '规则名称',
    rule_content TEXT NOT NULL COMMENT '规则内容',
    rule_type VARCHAR(20) NOT NULL COMMENT '规则类型：drools,easy_rules',
    status TINYINT DEFAULT 1 COMMENT '状态：0禁用,1启用',
    priority INT DEFAULT 0 COMMENT '优先级',
    tenant_id BIGINT COMMENT '租户ID',
    created_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

## 总结

推荐的技术组合能够满足不同复杂度的业务需求，既保证了性能，又具备良好的扩展性。通过模块化设计，可以根据实际需要逐步集成各种引擎，降低系统复杂度和维护成本。
