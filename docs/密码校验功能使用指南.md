## 密码校验功能使用指南

### 一、触发场景（推荐集成位置）

#### 1.1 用户注册场景

在用户注册时，通过PasswordValidationService验证密码强度。

**注册API示例**：
```java
@PostMapping("/register")
public CommonResult<Void> register(@Valid @RequestBody UserRegisterReqDTO reqDTO) {
    // 密码通过Bean Validation的@CheckPasswordField注解自动校验
    // 如果需要编程式额外校验，可以调用：
    // passwordValidationService.validatePassword(reqDTO.getPassword());
    
    // 继续注册流程...
    userService.register(reqDTO);
    return CommonResult.success();
}
```

#### 1.2 修改密码场景

在用户修改密码时，验证新密码的强度。

**修改密码API示例**：
```java
@PostMapping("/change-password")
@PreAuthorize("@ss.hasPermission('system:user:update-password')")
public CommonResult<Void> changePassword(@Valid @RequestBody ChangePasswordReqDTO reqDTO) {
    // 新密码通过@CheckPasswordField注解自动校验
    // 旧密码校验...
    // 新密码与旧密码不同检查...
    
    userService.changePassword(reqDTO.getOldPassword(), reqDTO.getNewPassword());
    return CommonResult.success();
}
```

#### 1.3 重置密码场景（管理员操作）

在管理员重置用户密码时，验证新密码的强度。

**重置密码API示例**：
```java
@PostMapping("/reset-password")
@PreAuthorize("@ss.hasPermission('system:user:reset-password')")
public CommonResult<Void> resetPassword(@Valid @RequestBody ResetPasswordReqDTO reqDTO) {
    // 新密码通过@CheckPasswordField注解自动校验
    
    userService.resetPassword(reqDTO.getUserId(), reqDTO.getNewPassword());
    return CommonResult.success();
}
```

### 二、集成方式

#### 2.1 方式一：注解方式（推荐）

最简单、最优雅的方式。配合Bean Validation框架使用。

**DTO示例**：
```java
@Data
@AllArgsConstructor
@NoArgsConstructor
public class UserRegisterReqDTO {
    
    @NotEmpty(message = "用户名不能为空")
    @Length(min = 3, max = 20, message = "用户名长度3-20字符")
    private String username;
    
    @NotEmpty(message = "密码不能为空")
    @CheckPasswordField  // 自动进行密码强度校验
    private String password;
    
    @NotEmpty(message = "邮箱不能为空")
    @Email(message = "邮箱格式不正确")
    private String email;
}
```

**Controller示例**：
```java
@PostMapping("/register")
public CommonResult<Void> register(@Valid @RequestBody UserRegisterReqDTO reqDTO) {
    // @Valid注解会自动触发@CheckPasswordField的校验
    // 如果密码不符合要求，会抛出MethodArgumentNotValidException
    
    userService.register(reqDTO);
    return CommonResult.success();
}
```

#### 2.2 方式二：编程式调用

需要在业务逻辑中主动调用校验。

**Service示例**：
```java
@Service
public class UserRegisterServiceImpl {
    
    @Resource
    private PasswordValidationService passwordValidationService;
    
    public void register(UserRegisterReqDTO reqDTO) {
        // 主动调用密码校验
        try {
            passwordValidationService.validatePassword(reqDTO.getPassword());
        } catch (WeakPasswordException e) {
            throw exception(USER_PASSWORD_WEAK, e.getMessage());
        }
        
        // 继续处理注册流程...
    }
}
```

#### 2.3 方式三：跨模块Feign调用

其他模块需要校验密码时，通过Feign调用。

**调用示例**：
```java
@Service
public class ExternalServiceImpl {
    
    @Resource
    private PasswordValidationApi passwordValidationApi;
    
    public void createUser(UserCreateReqDTO reqDTO) {
        // 通过Feign调用密码校验API
        CommonResult<Void> result = passwordValidationApi.validatePassword(reqDTO.getPassword());
        if (!result.isSuccess()) {
            throw new ServiceException(result.getMsg());
        }
        
        // 继续处理...
    }
}
```

### 四、异常处理

#### 4.1 在全局异常处理器中处理

```java
@RestControllerAdvice
public class GlobalExceptionHandler {
    
    /**
     * 处理弱密码异常
     */
    @ExceptionHandler(WeakPasswordException.class)
    public ResponseEntity<CommonResult<?>> handleWeakPasswordException(WeakPasswordException e) {
        log.warn("密码强度校验失败: {}", e.getMessage());
        return ResponseEntity.ok(CommonResult.error("密码强度不符合要求: " + e.getMessage()));
    }
    
    /**
     * 处理Bean Validation异常（包含@CheckPasswordField校验失败）
     */
    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<CommonResult<?>> handleMethodArgumentNotValidException(MethodArgumentNotValidException e) {
        BindingResult result = e.getBindingResult();
        String message = result.getFieldErrors().stream()
                .map(FieldError::getDefaultMessage)
                .collect(Collectors.joining("; "));
        log.warn("请求参数校验失败: {}", message);
        return ResponseEntity.ok(CommonResult.error("请求参数不合法: " + message));
    }
}
```

### 五、配置管理

#### 5.1 租户管理员配置密码策略

租户管理员可以通过UI接口配置以下密码策略参数：

**配置参数说明**：

| 参数名 | 类型 | 必填 | 说明 | 示例值 |
|--------|------|-----|------|--------|
| enableWeakPassword | BOOLEAN | 是 | 是否启用弱密码校验 | true |
| minLength | INTEGER | 是 | 最小长度 | 8-32之间 |
| extraCharacter | JSON[STRING] | 是 | 额外字符类型 | "uppperCase,specialChar" |

**配置更新API**：
```
POST /infra/security-config/batch-update
{
  "configs": [
    {"configKey": "enableWeakPassword", "configValue": "true"},
    {"configKey": "minLength", "configValue": "12"},
    {"configKey": "extraCharacter", "configValue": "uppperCase,specialChar"}
  ]
}
```

#### 5.2 配置生效时间

- 配置修改**立即生效**
- Redis缓存会自动清除
- 下一次密码校验会使用最新配置

### 六、常见问题

#### 6.1 Q: 什么时候触发密码校验？

A: 在以下场景触发：
- 用户注册时
- 用户修改密码时
- 管理员重置用户密码时
- 其他需要校验密码强度的场景

#### 6.2 Q: 如果租户禁用了弱密码校验怎么办？

A: 如果租户设置 `enableWeakPassword = false`，则：
- 密码校验服务会直接返回，不进行任何校验
- 用户可以使用任意密码

#### 6.3 Q: 密码要求是固定的吗？

A: 不是。当前采用3级标准（最严）的固定规则，但以下参数**可配置**：
- `minLength` - 最小长度
- `extraCharacter` - 是否需要大写字母和特殊符号

其他校验规则（如键盘连续、逻辑连续等）目前固定为3级标准。

#### 6.4 Q: 跨模块调用时如何处理异常？

A: 通过Feign调用时，如果密码校验失败，API会返回错误信息：
```json
{
  "code": "密码校验错误码",
  "message": "密码强度不符合要求：具体失败原因",
  "data": null
}
```

调用方应该检查返回结果并相应处理。

### 七、性能考虑

#### 7.1 缓存优化

- 密码策略配置使用Redis缓存，TTL=30分钟
- 同一租户的重复校验无需重复查库
- 缓存自动清除机制确保配置及时更新

#### 7.2 校验性能

- 单次校验耗时：< 5ms（纯算法，无IO）
- 支持高并发场景
- 建议在应用层（而非数据库层）进行校验

### 八、安全建议

1. **不要在日志中输出密码**：已通过异常message处理，只输出失败原因
2. **通过HTTPS传输**：所有密码校验API都应使用HTTPS
3. **权限控制**：配置修改API需要管理员权限
4. **备份密码**：密码修改前应验证旧密码
5. **失败重试限制**：建议在用户层面实现登录失败重试限制
