# 密码校验功能 - 快速参考卡

## 🚀 30秒快速开始

### 最简单的方式：添加注解

```java
@Data
public class RegisterDTO {
    @CheckPasswordField  // ← 这一行就够了！
    private String password;
}

@PostMapping("/register")
public CommonResult<Void> register(@Valid @RequestBody RegisterDTO dto) {
    userService.register(dto);
    return CommonResult.success();
}
```

---

## 📚 三种使用方式

### 方式1️⃣：注解方式（推荐 ⭐⭐⭐⭐⭐）

```java
@CheckPasswordField
@NotEmpty(message = "密码不能为空")
private String password;
```

**优点**：最简洁、自动集成、无侵入
**缺点**：无
**场景**：所有场景

---

### 方式2️⃣：编程式（灵活 ⭐⭐⭐⭐）

```java
@Resource
private PasswordValidationService passwordValidationService;

public void changePassword(String newPassword) {
    try {
        passwordValidationService.validatePassword(newPassword);
        // 校验通过，继续处理...
    } catch (WeakPasswordException e) {
        throw new ServiceException("密码不符合要求: " + e.getMessage());
    }
}
```

**优点**：可处理异常、逻辑灵活
**缺点**：代码稍多
**场景**：需要自定义处理异常的场景

---

### 方式3️⃣：Feign跨模块（规范 ⭐⭐⭐⭐）

```java
@Resource
private PasswordValidationApi passwordValidationApi;

public void createUser(String password) {
    CommonResult<Void> result = passwordValidationApi.validatePassword(password);
    if (!result.isSuccess()) {
        throw new ServiceException(result.getMsg());
    }
    // 继续处理...
}
```

**优点**：跨模块解耦、规范统一
**缺点**：有网络IO开销
**场景**：跨模块调用

## ✅ 密码要求

### 必须满足的条件

| 要求 | 说明 | 示例 |
|------|------|------|
| 长度 | 8-32字符（可配置minLength） | MyPassword123! |
| 大小写字母 | 必须有大写和小写 | 禁用：abc123、ABC123 |
| 数字 | 必须至少有一个数字 | 必需：123 |
| 特殊符号 | 需要时必须有 | !@#$%^&*等 |
| 无键盘连续 | 横向/斜向最多2个 | 禁用：asdf、qwer、qaz |
| 无逻辑连续 | 最多2个 | 禁用：abc、123、xyz |
| 无重复字符 | 最多2个连续 | 禁用：aaaa、1111 |

### ❌ 弱密码示例

| 密码 | 原因 |
|------|------|
| password | 无数字、无特殊符号 |
| MyPassword | 无数字 |
| MyPassword123 | 无特殊符号 |
| Mypassword123! | ✅ 符合要求 |
| asdfgh123! | 包含键盘连续（asdf） |
| abcdef123! | 包含逻辑连续（abc） |

## 🔍 错误处理

### 常见错误码

| 错误码 | 含义 | 解决方案 |
|--------|------|---------|
| 1-001-005-000 | 密码长度过短 | 增加长度，至少{minLength}字符 |
| 1-001-005-001 | 缺少数字 | 添加至少一个数字 |
| 1-001-005-002 | 缺少字母 | 添加至少一个字母 |
| 1-001-005-003 | 缺少小写字母 | 添加至少一个小写字母 |
| 1-001-005-004 | 缺少大写字母 | 添加至少一个大写字母 |
| 1-001-005-005 | 缺少特殊符号 | 添加特殊符号 |
| 1-001-005-006 | 键盘横向连续 | 避免连续输入相邻字符 |
| 1-001-005-007 | 键盘斜向连续 | 避免斜向连续输入 |
| 1-001-005-008 | 逻辑连续 | 避免abc、123等连续字符 |
| 1-001-005-009 | 重复字符 | 避免aaaa、1111等 |

### 异常处理示例

```java
try {
    passwordValidationService.validatePassword(password);
} catch (WeakPasswordException e) {
    // 返回给前端
    return CommonResult.error(e.getMessage());
}
```

## 🔐 安全建议

✅ **务必做这些**：
- [ ] 使用HTTPS传输密码
- [ ] 不在日志中输出密码
- [ ] 密码修改前验证旧密码
- [ ] 实现登录失败重试限制
- [ ] 管理员有权修改策略

❌ **千万不要做这些**：
- [ ] 在日志中打印密码明文
- [ ] 在URL参数中传输密码
- [ ] 在错误信息中显示密码
- [ ] 将密码存储为明文

## 🆘 常见问题

**Q: 我配置了enableWeakPassword=false，还会校验吗？**
A: 不会。校验会直接返回。

**Q: 如果我改了minLength，现有的密码还能用吗？**
A: 可以。新规则只对新注册/修改密码时的新密码生效。

**Q: 特殊符号有哪些？**
A: `!"#$%&'()*+,-./:;<=>?@[\]^_`{|}~` (共32个)

**Q: 跨模块调用会很慢吗？**
A: 不会。通过Redis缓存后，典型场景<5ms。

**Q: 如何禁用某个租户的弱密码校验？**
A: 设置enableWeakPassword=false即可。
