# 后端插件扩展点示例及前后端协同方案

## 四个问题的解答（前后端协同）

1. **平台组件如何使用后端插件结果？**
   - 前端调用平台已有 API（如 `/api/formula/evaluate`），平台后端在公式/流水线中发现插件函数或处理器后调用插件，返回普通数据，前端用现有组件渲染即可。
   - 示例：前端上传图片 → 调用 `/api/formula/evaluate` → 后端调用插件函数 `OCR_RECOGNIZE` → 返回识别文本 → 前端用文本框显示。

2. **自定义 UI 场景是否需要前端插件？**
   - 只有当平台现有组件不能满足展示/交互需求时才需要（如地图选点、3D、复杂富文本、特殊仪表盘卡片）。
   - 若只是展示字符串/数字/表格/图表，通常不需要前端插件。

3. **“约定的 API 协议”指什么？**
   - 前端插件通过 SDK 调用后端插件：
     - `api.callFunction(name, ...args)`: 调用后端 `CustomFunction`。
     - `api.fetch(path, options)`: 调用后端 `HttpHandler` 暴露的 REST API。
     - `api.getRecords(table, query)`: 调用平台数据服务（封装的数据查询）。
   - 示例：地图选点组件点击地图后执行 `api.callFunction('GET_ADDRESS_BY_LATLNG', lat, lng)`，后端插件返回地址；搜索时执行 `api.fetch('/plugin/map/search?keyword=北京')`。

4. **Module Federation 是什么/为何使用？**
   - Webpack 5 微前端能力：允许主应用与远程插件在运行时共享依赖（如 React），动态加载插件而无需重建主应用，避免多版本冲突、减小体积。
   - 作用：插件独立构建、独立部署；主应用运行时按需加载插件组件，复用主应用的 React/@onebase/plugin-sdk。

## 四个扩展点示例

### 1) CustomFunction（自定义函数）——OCR 识别
```java
@Extension
public class OcrFunction implements CustomFunction {
    @Override
    public String name() { return "OCR_RECOGNIZE"; }
    @Override
    public String description() { return "图片文字识别"; }

    @Override
    public Object execute(PluginContext ctx, Object... args) throws Exception {
        String imageUrl = (String) args[0];
        // 伪代码：调用第三方 OCR
        String apiKey = ctx.getConfig("ocrApiKey");
        String text = doOcr(apiKey, imageUrl);
        return text;
    }
}
```
- 前端：调用 `/api/formula/evaluate` 或 `api.callFunction('OCR_RECOGNIZE', imageUrl)`，结果用文本组件渲染。

### 2) HttpHandler（HTTP 接口）——地址搜索
```java
@Extension
public class AddressSearchHandler implements HttpHandler {
    @Override
    public String pathPattern() { return "/map/search"; }
    @Override
    public String method() { return "GET"; }
    @Override
    public boolean requireAuth() { return true; }

    @Override
    public HttpResponse handle(PluginContext ctx, HttpRequest request) {
        String keyword = request.getParam("keyword");
        String apiKey = ctx.getConfig("mapApiKey");
        List<AddressResult> results = searchAddress(apiKey, keyword);
        return HttpResponse.ok(results);
    }
}
```
- 前端插件：`api.fetch('/plugin/map-plugin/map/search?keyword=北京')` 获取列表，用自定义组件展示。

### 3) DataProcessor（数据处理器）——敏感数据脱敏
```java
@Extension
public class DataMaskProcessor implements DataProcessor {
    @Override
    public String type() { return "DATA_MASK"; }
    @Override
    public Class<?> inputType() { return List.class; }
    @Override
    public Class<?> outputType() { return List.class; }
    @Override
    public String description() { return "手机号/身份证/银行卡脱敏"; }

    @Override
    @SuppressWarnings("unchecked")
    public Object process(PluginContext ctx, Object input, Map<String, Object> config) {
        List<Map<String, Object>> data = (List<Map<String, Object>>) input;
        Map<String, String> rules = (Map<String, String>) config.get("fields");
        List<Map<String, Object>> result = new ArrayList<>();
        for (Map<String, Object> row : data) {
            Map<String, Object> r = new HashMap<>(row);
            rules.forEach((field, type) -> {
                if (r.get(field) != null) {
                    r.put(field, mask(r.get(field).toString(), type));
                }
            });
            result.add(r);
        }
        return result;
    }

    private String mask(String v, String type) {
        return switch (type) {
            case "PHONE" -> v.length()==11 ? v.substring(0,3)+"****"+v.substring(7) : v;
            case "ID_CARD" -> v.length()>8 ? v.substring(0,3)+"***********"+v.substring(v.length()-4) : v;
            case "BANK_CARD" -> v.length()>8 ? v.substring(0,4)+" **** **** "+v.substring(v.length()-4) : v;
            case "EMAIL" -> v.contains("@") ? v.charAt(0)+"***"+v.substring(v.indexOf('@')) : v;
            default -> v;
        };
    }
}
```
- 平台调用：导出/同步前先 `processor.process(data, {fields: {phone:"PHONE", idCard:"ID_CARD"}})`。

### 4) EventListener（事件监听器）——数据变更通知
```java
@Extension
public class DataChangeNotifyListener implements EventListener {
    @Override
    public String[] eventTypes() { return new String[]{"DATA_CREATED","DATA_UPDATED","DATA_DELETED"}; }
    @Override
    public boolean async() { return true; }
    @Override
    public int order() { return 200; }
    @Override
    public String description() { return "数据变更通知（钉钉/企微）"; }

    @Override
    public void onEvent(PluginContext ctx, PluginEvent event) {
        Map<String, Object> data = event.getData();
        String table = (String) data.get("tableName");
        Map<String, Object> row = (Map<String, Object>) data.get("data");
        if (!"order".equals(table)) return;
        String msg = String.format("订单 %s 发生 %s", row.get("id"), event.getType());
        sendDingTalk(msg);
    }

    private void sendDingTalk(String msg) {
        // 伪代码：调用钉钉 webhook
        System.out.println("[DingTalk] " + msg);
    }
}
```
- 平台触发：数据服务 CRUD 后 `eventDispatcher.dispatch(new PluginEvent("DATA_CREATED", {...}))`，监听器异步通知。

## 前后端插件协同（协议与调用）

- 前端插件调用后端插件：
  - `api.callFunction('FUNC', ...args)` → 后端 `CustomFunction`。
  - `api.fetch('/plugin/{pluginId}/xxx', options)` → 后端 `HttpHandler`。
  - `api.getRecords(table, query)` → 平台数据服务封装。
- 何时仅前端调用数据服务？
  - 简单 CRUD/列表展示：前端直接用数据 API 或 `api.getRecords` 即可。
  - 涉及敏感数据、复杂聚合、外部 API、跨表业务：放在后端插件处理，通过 `HttpHandler`/`CustomFunction` 暴露。
- 何时需要前端插件？
  - 仅当平台现有组件不能满足 UI/交互需求（地图、3D、自定义控件、特殊仪表盘卡片）。
- Module Federation（前端插件加载）：
  - 让主应用与插件共享 React/SDK，运行时动态加载远程插件组件，避免版本冲突，插件独立发布。

## 快速索引
- **自定义函数**：`CustomFunction` → 公式/规则调用，返回结果直接渲染。
- **HTTP 接口**：`HttpHandler` → 暴露 REST，前端 `api.fetch` 调用。
- **数据处理器**：`DataProcessor` → ETL/脱敏/聚合，平台流水线/导入导出中使用。
- **事件监听器**：`EventListener` → 监听平台事件，做通知/同步/联动。
- **前端插件调用协议**：`api.callFunction` / `api.fetch` / `api.getRecords`。
- **何时需要前端插件**：现有组件不足以承载定制 UI 时（地图、3D、复杂控件）。
- **模块联邦作用**：微前端共享依赖，插件独立构建、运行时加载、避免多版本 React。
