## 📖 文档使用指南

### 新人入门
1. 阅读 [项目概述](./project-overview.md) 了解项目全貌
2. 学习 [开发指南](./development-guide.md) 掌握开发规范
3. 查看 [模块参考手册](./modules-reference.md) 理解模块功能
4. 根据 [学习计划](./learning-plan.md) 系统学习

### 开发人员
1. 参考 [开发指南](./development-guide.md) 进行开发
2. 查看 [API 接口契约](./api-contracts-backend.md) 了解接口规范
3. 遵循 [架构设计](./architecture-plugin-system.md) 进行架构决策
4. 阅读 [ADR 文档](#-架构决策记录-adr) 了解历史决策

### 项目经理
1. 查看 [Epic 清单](./epics.md) 了解开发计划
2. 阅读 [PRD](./prd.md) 了解产品需求
3. 查看 [实施就绪报告](#-项目管理) 评估项目状态
4. 参考 [项目计划重评估](./project-plan-reassessment.md) 了解进度

### 测试人员
1. 查看 [测试计划](./test-plan-plugin-system.md) 编写测试用例
2. 参考 [测试报告](#-测试文档) 了解测试情况

## 📝 文档贡献

### 文档更新规范
1. 保持文档与代码同步更新
2. 使用清晰的标题层级
3. 提供代码示例和使用说明
4. 标注最后更新时间
5. 重要决策记录在 ADR 中

### 文档命名规范
- 英文文档: `kebab-case.md`
- 中文文档: `中文标题.md`
- 会议记录: `session-YYYY-MM-DD-{主题}.md`
- ADR: `adr-{编号}-{主题}.md`
