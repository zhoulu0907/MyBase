# 权限查询框架使用指南

## 一、架构概述

权限查询框架提供了在查询时应用数据权限和字段权限过滤的完整解决方案，主要包括：

1. **DataPermissionFilterBuilder** - 数据权限过滤条件构建器
2. **FieldPermissionFilter** - 字段权限过滤器
3. **PermissionQueryHelper** - 权限查询辅助类（统一入口）
4. **AbstractMetadataDataMethodCoreService** - 抽象类中提供的辅助方法

## 二、使用场景

### 场景1：单条数据查询（GET操作）

在 `getData` 方法中应用权限过滤：

```java
@Override
protected Map<String, Object> getData(ProcessContext context) {
    Long entityId = context.getEntity().getId();
    Object id = context.getId();
    
    // 1. 准备查询
    MetadataBusinessEntityDO entity = validateEntityExists(entityId);
    List<MetadataEntityFieldDO> fields = getEntityFields(entityId);
    MetadataDatasourceDO datasource = metadataDatasourceCoreService.getDatasource(entity.getDatasourceId());
    
    AnylineService<?> temporaryService = temporaryDatasourceService.createTemporaryService(datasource);
    log.info("成功切换到数据源：{}", datasource.getCode());
    
    return TenantUtils.executeIgnore(() -> {
        // 2. 查询数据
        Map<String, Object> resultData = queryDataByIdWithService(
                temporaryService, 
                quoteTableName(entity.getTableName()), 
                id, 
                fields
        );
        
        if (resultData == null || resultData.isEmpty()) {
            throw exception(BUSINESS_ENTITY_NOT_EXISTS);
        }
        
        // 3. 应用字段权限过滤（移除无权读取的字段）
        resultData = filterQueryResultFields(resultData, context);
        
        // 4. 构建响应
        Map<String, Object> response = buildDataResponse(entity, resultData, fields);
        context.setProcessedData(response);
        
        return response;
    });
}
```

### 场景2：分页查询（GET_PAGE操作）

在 `queryData` 方法中应用权限过滤：

```java
protected Map<String, Object> queryData(ProcessContext context) {
    Long entityId = context.getEntity().getId();
    Map<String, Object> conditions = context.getData();
    Integer pageNo = (Integer)conditions.get("pageNo");
    Integer pageSize = (Integer)conditions.get("pageSize");
    Map<String, Object> filters = (Map<String, Object>)conditions.get("filters");
    
    // 1. 准备查询
    MetadataBusinessEntityDO entity = validateEntityExists(entityId);
    List<MetadataEntityFieldDO> fields = getEntityFields(entityId);
    MetadataDatasourceDO datasource = metadataDatasourceCoreService.getDatasource(entity.getDatasourceId());
    
    AnylineService<?> temporaryService = temporaryDatasourceService.createTemporaryService(datasource);
    log.info("成功切换到数据源：{}", datasource.getCode());
    
    return TenantUtils.executeIgnore(() -> {
        // 2. 构建查询条件
        ConfigStore configs = new DefaultConfigStore();
        
        // 2.1 应用业务过滤条件
        if (filters != null && !filters.isEmpty()) {
            // ... 处理业务过滤条件 ...
        }
        
        // 2.2 应用数据权限过滤（重要！）
        applyQueryPermissionFilter(configs, context);
        
        // 2.3 设置排序和分页
        if (pageNo != null && pageSize != null) {
            PageNavi page = new DefaultPageNavi(pageNo, pageSize);
            configs.setPageNavi(page);
        }
        
        // 3. 执行查询
        long total = temporaryService.count(quoteTableName(entity.getTableName()), configs);
        DataSet dataSet = temporaryService.querys(quoteTableName(entity.getTableName()), configs);
        
        // 4. 转换查询结果
        List<Map<String, Object>> list = new ArrayList<>();
        for (int i = 0; i < dataSet.size(); i++) {
            DataRow row = dataSet.getRow(i);
            Map<String, Object> data = convertDataRowToMap(row, fields);
            list.add(buildDataResponse(entity, data, fields));
        }
        
        // 5. 应用字段权限过滤（移除无权读取的字段）
        list = filterQueryResultListFields(list, context);
        
        // 6. 构建响应
        Map<String, Object> result = new HashMap<>();
        result.put("list", list);
        result.put("total", total);
        context.setProcessedData(result);
        
        return result;
    });
}
```

### 场景3：完整示例 - MetadataDataMethodQueryImpl

这是一个完整的查询实现示例，展示了如何在实际业务中使用权限查询框架：

```java

@Component
public class MetadataDataMethodQueryImpl extends AbstractMetadataDataMethodCoreService {

    public void validateDataIntegrity(Map<String, Object> data, List<MetadataEntityFieldDO> fields) {
        // 查询操作无需校验
    }

    /**
     * 设置默认数据（查询操作无需设置默认值）
     */
    protected Map<String, Object> processDataAndSetDefaults(Map<String, Object> data, List<MetadataEntityFieldDO> fields) {
        return null;
    }

    /**
     * 权限校验使用父类的实现
     */
    // 无需重写 validatePermission 方法，使用父类的 permissionManager.checkPermission(context)

    /**
     * 数据校验（查询操作无需数据校验）
     */
    protected void validateData(ProcessContext context) {
        // 查询操作无需数据校验
    }

     * 存储数据（查询操作无需存储）
     */
    protected void storeData(ProcessContext context) {
        // 查询操作无需存储
    }

    /**
     * 单条数据查询（带权限过滤）
     */
    @Override
    protected Map<String, Object> getData(ProcessContext context) {
        Long entityId = context.getEntity().getId();
        Object id = context.getId();
        
        MetadataBusinessEntityDO entity = validateEntityExists(entityId);
        List<MetadataEntityFieldDO> fields = getEntityFields(entityId);
        MetadataDatasourceDO datasource = metadataDatasourceCoreService.getDatasource(entity.getDatasourceId());
        
        if (datasource == null) {
            throw exception(DATASOURCE_NOT_EXISTS);
        }
        
        AnylineService<?> temporaryService = temporaryDatasourceService.createTemporaryService(datasource);
        log.info("成功切换到数据源：{}", datasource.getCode());
        
        TenantUtils.executeIgnore(() -> {
            // 查询主表数据
            Map<String, Object> resultData = queryDataByIdWithService(
                    temporaryService, 
                    quoteTableName(entity.getTableName()), 
                    id, 
                    fields
            );
            
            if (resultData == null || resultData.isEmpty()) {
                throw exception(BUSINESS_ENTITY_NOT_EXISTS);
            }
            
            // 【关键】应用字段权限过滤
            resultData = filterQueryResultFields(resultData, context);
            
            // 构建响应数据
            Map<String, Object> response = buildDataResponse(entity, resultData, fields);
            context.setProcessedData(response);
        });
        
        // 查询子表数据（如果有关联）
        querySubTableData(context, entityId, id);
        
        return context.getProcessedData();
    }

    /**
     * 分页查询（带权限过滤）
     */
    protected Map<String, Object> queryData(ProcessContext context) {
        Long entityId = context.getEntity().getId();
        Map<String, Object> conditions = context.getData();
        
        Integer pageNo = (Integer) conditions.get("pageNo");
        Integer pageSize = (Integer) conditions.get("pageSize");
        String sortField = (String) conditions.get("sortField");
        String sortDirection = (String) conditions.get("sortDirection");
        Map<String, Object> filters = (Map<String, Object>) conditions.get("filters");
        
        log.info("分页查询参数 - entityId: {}, pageNo: {}, pageSize: {}", entityId, pageNo, pageSize);
        
        MetadataBusinessEntityDO entity = validateEntityExists(entityId);
        List<MetadataEntityFieldDO> fields = getEntityFields(entityId);
        MetadataDatasourceDO datasource = metadataDatasourceCoreService.getDatasource(entity.getDatasourceId());
        
        if (datasource == null) {
            throw exception(DATASOURCE_NOT_EXISTS);
        }
        
        AnylineService<?> temporaryService = temporaryDatasourceService.createTemporaryService(datasource);
        log.info("成功切换到数据源：{}", datasource.getCode());
        
        boolean hasDeletedField = fields.stream()
                .anyMatch(f -> "deleted".equalsIgnoreCase(f.getFieldName()));
        
        return TenantUtils.executeIgnore(() -> {
            // 1. 构建查询条件
            ConfigStore configs = new DefaultConfigStore();
            
            // 1.1 处理业务过滤条件
            if (filters != null && !filters.isEmpty()) {
                applyBusinessFilters(configs, filters, fields);
            }
            
            // 1.2 处理软删除
            if (hasDeletedField) {
                configs.and(Compare.EQUAL, "deleted", 0);
            }
            
            // 1.3 【关键】应用数据权限过滤
            applyQueryPermissionFilter(configs, context);
            
            // 1.4 处理排序
            if (StringUtils.hasText(sortField)) {
                String orderClause = sortField + ("desc".equalsIgnoreCase(sortDirection) ? " DESC" : " ASC");
                configs.order(orderClause);
            } else {
                String primaryKeyField = getPrimaryKeyFieldName(fields);
                configs.order(primaryKeyField + " DESC");
            }
            
            // 1.5 处理分页
            if (pageNo != null && pageSize != null) {
                PageNavi page = new DefaultPageNavi(pageNo, pageSize);
                configs.setPageNavi(page);
            }
            
            // 2. 执行查询
            long total = temporaryService.count(quoteTableName(entity.getTableName()), configs);
            DataSet dataSet = temporaryService.querys(quoteTableName(entity.getTableName()), configs);
            
            // 3. 转换查询结果
            List<Map<String, Object>> list = new ArrayList<>();
            for (int i = 0; i < dataSet.size(); i++) {
                DataRow row = dataSet.getRow(i);
                Map<String, Object> data = convertDataRowToMap(row, fields);
                list.add(buildDataResponse(entity, data, fields));
            }
            
            // 4. 【关键】应用字段权限过滤
            list = filterQueryResultListFields(list, context);
            
            // 5. 构建响应
            Map<String, Object> result = new HashMap<>();
            result.put("list", list);
            result.put("total", total);
            context.setProcessedData(result);
            
            return result;
        });
    }

    /**
     * 应用业务过滤条件
     */
    private void applyBusinessFilters(ConfigStore configs, 
                                       Map<String, Object> filters, 
                                       List<MetadataEntityFieldDO> fields) {
        Set<String> fieldNames = fields.stream()
                .map(MetadataEntityFieldDO::getFieldName)
                .collect(Collectors.toSet());
        
        for (Map.Entry<String, Object> entry : filters.entrySet()) {
            String key = entry.getKey();
            Object value = entry.getValue();
            
            if (value == null || "deleted".equalsIgnoreCase(key) || "tenant_id".equalsIgnoreCase(key)) {
                continue;
            }
            
            if (value instanceof Map) {
                // 结构化过滤条件
                @SuppressWarnings("unchecked")
                Map<String, Object> cond = (Map<String, Object>) value;
                String fieldName = (String) cond.getOrDefault("fieldName", key);
                Object fieldValue = cond.get("value");
                String operator = (String) cond.getOrDefault("operator", "CONTAINS");
                
                if (fieldValue != null && fieldNames.contains(fieldName)) {
                    applyOperatorCondition(configs, fieldName, operator, fieldValue);
                }
            } else {
                // 简单过滤条件
                if (fieldNames.contains(key)) {
                    configs.and(Compare.LIKE, key, value);
                }
            }
        }
    }

    /**
     * 查询子表数据
     */
    private void querySubTableData(ProcessContext context, Long entityId, Object parentId) {
        // ... 子表查询逻辑 ...
        // 注意：子表数据也需要应用字段权限过滤
    }

    /**
     * 格式化结果
     */
    protected Map<String, Object> formatResult(ProcessContext context) {
        return context.getProcessedData();
    }
}
```

## 三、关键方法说明

### 3.1 applyQueryPermissionFilter(ConfigStore, ProcessContext)

**作用**：在查询前应用数据权限过滤

**使用时机**：在构建 ConfigStore 查询条件之后，执行查询之前调用

**效果**：
- 根据用户的数据权限配置，自动向 ConfigStore 添加 WHERE 条件
- 例如："本人提交" 会添加 `creator = 当前用户ID` 的条件
- 支持多种权限标签和权限级别的组合

### 3.2 filterQueryResultFields(Map, ProcessContext)

**作用**：过滤单条查询结果中的字段

**使用时机**：在查询后，构建响应之前调用

**效果**：
- 移除用户无权读取的字段
- 保留系统字段（如ID、created_at等）
- 返回过滤后的数据

### 3.3 filterQueryResultListFields(List, ProcessContext)

**作用**：批量过滤查询结果列表中的字段

**使用时机**：在查询后，构建响应之前调用

**效果**：
- 对列表中的每条数据应用字段权限过滤
- 适用于分页查询、列表查询等场景

## 四、权限过滤流程图

```
┌─────────────────────────────────────────────────┐
│           用户发起查询请求                         │
└─────────────┬───────────────────────────────────┘
              ↓
┌─────────────────────────────────────────────────┐
│  1. Controller 构建权限上下文                      │
│     - 调用 PermissionContextBuilder               │
│     - 设置到 RequestContext                       │
└─────────────┬───────────────────────────────────┘
              ↓
┌─────────────────────────────────────────────────┐
│  2. Service 执行操作权限校验                       │
│     - permissionManager.checkPermission()         │
│     - 校验是否有查询权限                           │
└─────────────┬───────────────────────────────────┘
              ↓
┌─────────────────────────────────────────────────┐
│  3. 构建查询条件（业务过滤 + 数据权限过滤）           │
│     - 添加业务过滤条件                             │
│     - applyQueryPermissionFilter() ← 关键步骤      │
│       └→ 根据数据权限添加 WHERE 条件                │
└─────────────┬───────────────────────────────────┘
              ↓
┌─────────────────────────────────────────────────┐
│  4. 执行数据库查询                                 │
│     - temporaryService.querys()                   │
│     - 只查询用户有权限访问的数据                    │
└─────────────┬───────────────────────────────────┘
              ↓
┌─────────────────────────────────────────────────┐
│  5. 过滤查询结果字段                               │
│     - filterQueryResultFields() ← 关键步骤         │
│       └→ 移除用户无权读取的字段                     │
└─────────────┬───────────────────────────────────┘
              ↓
┌─────────────────────────────────────────────────┐
│  6. 返回过滤后的数据                               │
└─────────────────────────────────────────────────┘
```

## 五、注意事项

### 5.1 性能考虑

1. **数据权限过滤在 SQL 层面**：
   - 通过 ConfigStore 添加 WHERE 条件，在数据库层面过滤数据
   - 避免了查询全部数据后再过滤的性能问题

2. **字段权限过滤在应用层面**：
   - 在查询结果返回后，在应用层移除无权限的字段
   - 对性能影响较小

3. **避免重复过滤**：
   - 单条查询只调用 `filterQueryResultFields`
   - 列表查询只调用 `filterQueryResultListFields`
   - 不要同时调用两者

### 5.2 权限校验顺序

正确的权限校验顺序：

1. **操作权限校验**（validatePermission）
   - 首先检查用户是否有查询权限
   - 如果没有权限，直接抛出异常，不执行查询

2. **数据权限过滤**（applyQueryPermissionFilter）
   - 在查询时过滤数据范围
   - 确保用户只能查询有权限的数据行

3. **字段权限过滤**（filterQueryResultFields）
   - 在查询后过滤字段
   - 确保用户只能看到有权限的字段

### 5.3 子表数据权限

查询子表数据时也需要应用权限过滤：

```java
// 查询子表数据
List<MetadataEntityFieldDO> subFields = getEntityFields(subEntityId);
DataSet subDataSet = temporaryService.querys(subTableName, subConfig);

List<Map<String, Object>> subList = new ArrayList<>();
for (int i = 0; i < subDataSet.size(); i++) {
    DataRow row = subDataSet.getRow(i);
    Map<String, Object> subData = convertDataRowToMap(row, subFields);
    
    // 应用字段权限过滤
    if (context.getMetadataPermissionContext() != null) {
        subData = permissionQueryHelper.filterQueryResult(
                subData,
                context.getMetadataPermissionContext(),
                subFields
        );
    }
    
    subList.add(subData);
}
```

### 5.4 系统内部调用

对于系统内部调用（无需权限校验），不设置 permissionContext：

```java
MetadataDataMethodRequestContext requestContext = new MetadataDataMethodRequestContext();
requestContext.setEntityId(entityId);
requestContext.setData(data);
requestContext.setMetadataDataMethodOpEnum(MetadataDataMethodOpEnum.GET);
// 不设置 permissionContext，权限过滤将自动跳过

Map<String, Object> result = dataMethodService.executeProcess(requestContext);
```
