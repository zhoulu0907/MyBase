# 权限校验使用示例

## 一、基本使用流程

### 1.1 在 Controller 层构建权限上下文

```java
@RestController
@RequestMapping("/metadata/data")
public class MetadataDataController {
    
    @Resource
    private PermissionContextBuilder permissionContextBuilder;
    
    @PostMapping("/create")
    public CommonResult<?> createData(@Valid @RequestBody CreateDataReqVO reqVO) {
        // 1. 获取当前登录用户信息
        LoginUserCtx loginUserCtx = new LoginUserCtx();
        loginUserCtx.setUserId(SecurityFrameworkUtils.getLoginUserId());
        loginUserCtx.setApplicationId(reqVO.getApplicationId());
        loginUserCtx.setToken(SecurityFrameworkUtils.getLoginUserToken());
        
        // 2. 构建权限上下文
        MetadataPermissionContext permissionContext = permissionContextBuilder
                .buildPermissionContext(loginUserCtx, reqVO.getMenuId(), reqVO.getEntityId());
        
        // 3. 构建请求上下文并设置权限信息
        MetadataDataMethodRequestContext requestContext = new MetadataDataMethodRequestContext();
        requestContext.setEntityId(reqVO.getEntityId());
        requestContext.setData(reqVO.getData());
        requestContext.setMethodCode(reqVO.getMethodCode());
        requestContext.setMenuId(reqVO.getMenuId());
        requestContext.setLoginUserCtx(loginUserCtx);
        requestContext.setPermissionContext(permissionContext); // 设置权限上下文
        requestContext.setMetadataDataMethodOpEnum(MetadataDataMethodOpEnum.CREATE);
        
        // 4. 执行业务逻辑（权限校验会自动执行）
        Map<String, Object> result = metadataDataMethodService.executeProcess(requestContext);
        
        return success(result);
    }
}
```

### 1.2 在 Service 层执行权限校验

```java
@Service
public class AbstractMetadataDataMethodCoreService {
    
    @Resource
    protected PermissionManager permissionManager;
    
    public Map<String, Object> doExecuteProcess(MetadataDataMethodRequestContext requestContext) {
        try {
            // 1. 校验实体存在
            MetadataBusinessEntityDO entity = validateEntityExists(entityId);
            
            // 2. 校验字段列表存在
            List<MetadataEntityFieldDO> fields = getEntityFields(entityId);
            
            // 3. 初始化上下文
            ProcessContext context = initializeContext(entity, fields, requestContext);
            
            // 4. 请求数据完整性校验
            validateDataIntegrity(requestContext.getData(), fields);
            
            // 5. 处理数据并设置默认值
            Map<String, Object> processedData = processDataAndSetDefaults(requestContext.getData(), fields);
            context.setProcessedData(processedData);
            
            // 6. 权限校验 - 自动按照优先级执行所有权限检查
            validatePermission(context);
            
            // 7. 数据校验
            validateData(context);
            
            // 8. 执行业务逻辑...
            
            return result;
        } catch (PermissionDeniedException e) {
            log.error("权限校验失败：{}", e.getMessage());
            throw e;
        }
    }
    
    protected void validatePermission(ProcessContext context) {
        log.info("开始执行权限校验：entityId={}, operationType={}", 
                context.getEntityId(), 
                context.getOperationType());
        
        // 使用权限管理器执行完整的权限校验流程
        // 会依次执行：操作权限 -> 数据权限 -> 字段权限
        permissionManager.checkPermission(context);
        
        log.info("权限校验完成：entityId={}", context.getEntityId());
    }
}
```

## 二、高级使用场景

### 2.1 快速检查单个操作权限

如果只需要检查用户是否有某个操作权限，可以使用便捷方法：

```java
@Service
public class SomeService {
    
    @Resource
    private PermissionManager permissionManager;
    
    public void doSomething(MetadataPermissionContext permissionContext) {
        // 快速检查是否有新增权限
        boolean canCreate = permissionManager.hasOperationPermission(
                permissionContext, "CREATE");
        
        if (!canCreate) {
            throw new PermissionDeniedException("操作权限", "CREATE", "无新增权限");
        }
        
        // 执行业务逻辑...
    }
}
```

### 2.2 自定义权限校验器

如果需要添加特殊的权限校验逻辑，可以实现自定义权限校验器：

```java
@Slf4j
@Component
public class WorkflowPermissionChecker implements PermissionChecker {
    
    @Override
    public String getPermissionType() {
        return "工作流权限";
    }
    
    @Override
    public boolean supports(ProcessContext context) {
        // 只在有工作流的实体上执行此检查
        return context.getEntity().getHasWorkflow() == 1;
    }
    
    @Override
    public void check(ProcessContext context) {
        // 检查用户是否有启动工作流的权限
        if (context.getOperationType() == MetadataDataMethodOpEnum.CREATE) {
            checkWorkflowStartPermission(context);
        }
        
        log.debug("工作流权限校验通过");
    }
    
    @Override
    public int getOrder() {
        return 40; // 在标准权限之后执行
    }
    
    private void checkWorkflowStartPermission(ProcessContext context) {
        // 实现工作流启动权限检查逻辑
        // ...
    }
}
```

### 2.3 跳过权限校验

在某些特殊场景下（如系统内部调用），可能需要跳过权限校验：

```java
@Service
public class SystemService {
    
    public void systemInternalOperation(Long entityId, Map<String, Object> data) {
        MetadataDataMethodRequestContext requestContext = new MetadataDataMethodRequestContext();
        requestContext.setEntityId(entityId);
        requestContext.setData(data);
        requestContext.setMetadataDataMethodOpEnum(MetadataDataMethodOpEnum.CREATE);
        // 不设置 permissionContext，权限校验将自动跳过
        
        metadataDataMethodService.executeProcess(requestContext);
    }
}
```

## 三、常见错误处理

### 3.1 捕获权限异常

```java
@RestController
public class DataController {
    
    @PostMapping("/create")
    public CommonResult<?> createData(@Valid @RequestBody CreateDataReqVO reqVO) {
        try {
            Map<String, Object> result = dataService.createData(reqVO);
            return success(result);
        } catch (PermissionDeniedException e) {
            log.warn("权限校验失败：type={}, permission={}, message={}", 
                    e.getPermissionType(), 
                    e.getDeniedPermission(), 
                    e.getMessage());
            return error(ErrorCodeConstants.PERMISSION_DENIED, e.getMessage());
        }
    }
}
```

### 3.2 全局异常处理

```java
@RestControllerAdvice
public class GlobalExceptionHandler {
    
    @ExceptionHandler(PermissionDeniedException.class)
    public CommonResult<?> handlePermissionDeniedException(PermissionDeniedException e) {
        log.warn("权限拒绝：{}", e.getMessage());
        
        // 构建友好的错误提示
        String message = String.format("您没有%s权限：%s", 
                e.getPermissionType(), 
                e.getMessage());
        
        return CommonResult.error(ErrorCodeConstants.PERMISSION_DENIED, message);
    }
}
```

## 五、性能优化建议

### 5.1 权限缓存

```java
@Service
public class CachedPermissionContextBuilder extends PermissionContextBuilder {
    
    @Resource
    private RedisTemplate<String, MetadataPermissionContext> redisTemplate;
    
    @Override
    public MetadataPermissionContext buildPermissionContext(
            LoginUserCtx loginUserCtx, Long menuId, Long entityId) {
        
        String cacheKey = String.format("permission:%s:%s:%s", 
                loginUserCtx.getUserId(), 
                loginUserCtx.getApplicationId(), 
                menuId);
        
        // 尝试从缓存获取
        MetadataPermissionContext cached = redisTemplate.opsForValue().get(cacheKey);
        if (cached != null) {
            log.debug("从缓存加载权限：key={}", cacheKey);
            return cached;
        }
        
        // 从数据库加载
        MetadataPermissionContext context = super.buildPermissionContext(
                loginUserCtx, menuId, entityId);
        
        // 写入缓存（30分钟过期）
        redisTemplate.opsForValue().set(cacheKey, context, 30, TimeUnit.MINUTES);
        
        return context;
    }
}
```

### 5.2 批量查询的数据权限过滤

```java
@Service
public class DataQueryService {
    
    public List<Map<String, Object>> queryDataList(
            Long entityId, MetadataPermissionContext permissionContext) {
        
        // 构建数据权限的 SQL 条件
        String whereClause = buildDataPermissionWhereClause(permissionContext);
        
        // 在 SQL 层面过滤数据，而不是查询后逐条校验
        String sql = String.format(
                "SELECT * FROM %s WHERE %s", 
                tableName, 
                whereClause
        );
        
        return executeQuery(sql);
    }
    
    private String buildDataPermissionWhereClause(
            MetadataPermissionContext permissionContext) {
        
        DataPermission dataPermission = permissionContext.getDataPermission();
        
        if (dataPermission.isAllAllowed()) {
            return "1=1"; // 允许查询所有数据
        }
        
        if (dataPermission.isAllDenied()) {
            return "1=0"; // 不允许查询任何数据
        }
        
        // 根据数据权限组构建 WHERE 条件
        List<String> conditions = new ArrayList<>();
        for (DataPermissionGroup group : dataPermission.getGroups()) {
            conditions.add(buildGroupCondition(group));
        }
        
        return "(" + String.join(" OR ", conditions) + ")";
    }
}
```

## 六、注意事项

1. **权限上下文构建**：
   - 必须在 Controller 层构建权限上下文
   - 确保 LoginUserCtx 包含正确的用户信息

2. **性能考虑**：
   - 使用 Redis 缓存权限信息
   - 批量查询时在 SQL 层面过滤数据权限
   - 避免在循环中重复校验权限

3. **异常处理**：
   - 统一捕获 PermissionDeniedException
   - 提供友好的错误提示
   - 记录详细的权限校验失败日志

4. **测试覆盖**：
   - 为每个权限校验器编写单元测试
   - 测试各种边界情况和异常场景
   - 进行集成测试验证完整流程

5. **安全性**：
   - 不要在错误消息中泄露敏感信息
   - 记录所有权限校验失败的日志
   - 定期审查权限配置
