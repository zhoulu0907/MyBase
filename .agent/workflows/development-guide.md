# 开发流程指南

## 开发流程

### 1. 任务分析
- **理解需求**: 仔细阅读需求描述或 Bug 报告
- **复述需求**: 用自己的话复述需求，确认理解正确
- **识别影响**: 确定受影响的模块和组件
- **评估风险**: 识别技术难点和潜在风险
- **制定方案**: 基于最佳实践设计实现方案

### 2. 代码实现
- **遵循规范**: 参考 `/code-generation-guide` 生成代码
- **小步提交**: 完成一个功能点就提交，避免大批量修改
- **代码审查**: 自我审查代码质量和规范

### 3. 测试验证
- **单元测试**: 参考 `/testing-guide` 编写测试
- **本地验证**: 启动应用，手动测试关键路径
- **边界测试**: 验证边界条件和异常情况

## 调试技巧

### 日志调试
**启用调试日志**:
在 `application-local.yml` 中设置：
```yaml
logging:
  level:
    com.cmsr.onebase: DEBUG
```

**关键日志点**:
- 方法入口和出口
- 关键业务分支
- 异常捕获点
- 外部调用（数据库、Redis、HTTP）

### 常见问题排查

#### 启动失败
1. 检查端口占用: `netstat -ano | findstr :8080`
2. 验证数据库连接: 检查配置和网络
3. 查看依赖冲突: `mvn dependency:tree`
4. 检查 Bean 循环依赖: 查看启动日志

#### 请求异常
- **400 Bad Request**: 检查参数校验注解（`@Valid`, `@NotNull`）
- **401/403**: 检查安全配置和权限
- **404 Not Found**: 验证 `@RequestMapping` 路径
- **500 Internal Error**: 查看详细异常堆栈

#### 性能问题
- 检查数据库慢查询: 开启 SQL 日志
- 分析线程状态: `jstack <pid>`
- 监控内存使用: `jmap -heap <pid>`
- 检查外部服务调用超时

## 性能优化

### 数据库优化
- 为常用查询字段添加索引
- 避免 N+1 查询（使用 JOIN 或批量查询）
- 使用分页查询大数据集
- 控制事务范围，避免长事务

### 缓存策略
- **本地缓存**: Caffeine（小数据量、高频访问）
- **分布式缓存**: Redis（共享数据、大数据量）
- **缓存更新**: 写操作时清除或更新缓存
- **缓存穿透**: 缓存空值或使用布隆过滤器

### 代码优化
- 减少对象创建，复用重对象
- 使用 `StringBuilder` 处理字符串拼接
- 及时关闭资源（使用 try-with-resources）
- 合理使用并发工具类

## 安全实践

### 输入验证
- 使用 `@Valid` 和校验注解（`@NotNull`, `@Size` 等）
- 验证数据格式、长度和范围
- 过滤特殊字符，防止注入攻击

### 敏感数据处理
- 密码使用 BCrypt 加密存储
- 日志中脱敏敏感字段（手机号、身份证）
- 配置文件中的密钥加密处理

### 权限控制
- 接口添加权限注解（`@PreAuthorize`）
- 验证当前用户的数据访问范围
- 防止越权访问（检查资源所有权）

## 常用命令

### Maven
```bash
# 完整构建（跳过测试）
mvn clean install -DskipTests -T 8

# 运行所有测试
mvn test

# 运行指定测试
mvn test -Dtest=ClassName

# 检查依赖更新
mvn versions:display-dependency-updates

# 格式化代码
mvn spotless:apply
```

## 性能优化建议

### 数据库优化
- 为查询字段添加索引
- 避免全表扫描
- 使用批量操作代替循环单条操作
- 控制事务范围，避免长事务

### 代码优化
- 减少对象创建，复用重对象
- 使用 StringBuilder 处理字符串拼接
- 及时关闭 IO 流和连接
- 使用并发工具类提高效率

### 缓存策略
- **本地缓存**: Caffeine / Guava (适用于小数据量、高频访问)
- **分布式缓存**: Redis (适用于共享数据、大数据量)
- **缓存一致性**: 注意缓存更新和失效策略
