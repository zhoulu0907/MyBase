# OneBase 3.0 信创数据库适配说明

## 背景
onebase3.0 主要基于 PostgreSQL 开发，为适配信创要求，需支持人大金仓（KingBase）和达梦（Dameng）数据库。

适配范围：DDL建表、DML初始化、Anyline框架、应用代码、SQL语句兼容。

---

## 人大金仓 (KingBase)

### 1. 数据库建表语句 DDL

#### 1.1 序列定义与使用
- 显式定义序列，表ID列不使用SERIAL类型
- 脚本位置：`sql/kingbase/onebase_cloud_v3_kingbase.sql`
- 序列使用示例：
  ```sql
  CREATE SEQUENCE IF NOT EXISTS seq_system_uid_worker_node;
  INSERT INTO system_role_menu (id, ...) VALUES (nextval('system_role_menu_seq'), ...);
  ```

#### 1.2 索引与兼容性
- 移除所有 `USING btree` 语法，直接使用 `CREATE INDEX`
- 保留 PostgreSQL 特性：int8/int4/int2、json类型、外键级联
- 表名和列名统一加双引号

#### 1.3 执行脚本
```bash
ksql -h localhost -p 54321 -U system -d onebase_cloud_v3 -f sql/kingbase/onebase_cloud_v3_kingbase.sql
```

### 2. 数据初始化脚本 DML

- 可复用 PostgreSQL 初始化脚本（需确认是否有专用 kingbase DML 脚本）
- 执行后必须同步序列值：
  ```sql
  SELECT setval('system_role_menu_seq', (SELECT COALESCE(MAX(id), 0) + 1 FROM system_role_menu));
  ```

### 3. Anyline 框架改造

#### 3.1 依赖配置
- 已引入：`anyline-data-jdbc-kingbase` + `kingbase8:8.6.0`
- 位置：`onebase-dependencies/pom.xml`、`onebase-framework/onebase-spring-boot-starter-orm/pom.xml`

#### 3.2 类型转换
- 使用 Anyline 原生转换机制，无需额外注册转换器
- 驱动自动识别：driver name 包含 "kingbase" 或 "人大金仓"

#### 3.3 日志配置
- `logback-spring.xml` 中配置了 KingBase 专用日志（DEBUG级别）

#### 3.4 AnyLineDBInfoListener 复杂字段序列化改造
- 位置：`onebase-framework/onebase-project-base/.../AnyLineDBInfoListener.java`
- 提交：SZHDZ-3329 (2025/11/18)
- 改造内容：
  - **检测逻辑**：`hasComplexFields()` 方法检测实体中带 `@Column` 注解的 Map/List 字段
  - **序列化转换**：`convertEntityToDataRowWithSerialization()` 将 Map/List 字段通过 JsonUtils 序列化为 JSON 字符串
  - **插入流程**：检测到复杂字段时，转换为 DataRow 并使用 AnylineService 手动插入，返回 SWITCH.BREAK 阻止原 Entity 插入
  - **降级处理**：转换失败时自动降级为原 Entity 插入流程
- 依赖新增：注入 AnylineService 以支持自定义插入操作
- 适用场景：解决实体字段为 Map/List 类型时，不同数据库对 JSON 存储支持差异的问题

### 4. 应用代码改造

#### 4.1 WorkerNodeDAO.addWorkerNode
- 位置：`onebase-framework/onebase-spring-boot-starter-orm/.../WorkerNodeDAO.java` (line 104-119)
- 改动：使用 `nextval('seq_system_uid_worker_node')` 显式调用序列生成ID
- 目的：解决不同数据库ID生成机制差异

#### 4.2 TemporaryDatasourceService
- 位置：`onebase-module-data/.../TemporaryDatasourceService.java` (line 505-527)
- KingBase 测试查询：`SELECT 1`（与 PostgreSQL 相同）

#### 4.3 序列冲突修复
- 参考：`sql/patch/fix_system_uid_worker_node_emergency.sql`
- 需根据 KingBase 语法调整后执行

## 达梦 (Dameng)

### 1. 数据库建表语句 DDL

#### 1.1 数据类型映射
- 脚本位置：`sql/dameng/onebase_cloud_v3_dm.sql`
- 已完成转换（累计1762处兼容性修复）：
  - int8 → BIGINT (988处)
  - int4 → INTEGER
  - int2 → SMALLINT
  - bytea → BLOB
  - bpchar → CHAR
  - 移除 `::character varying` 等类型转换语法 (150处)

#### 1.2 自增列实现
- 部分表使用 `BIGINT IDENTITY(1,1)` (8处)：
  - system_role_menu、system_user_post、system_user_role
  - metadata_data_system_method、metadata_component_field_type
  - metadata_validation_type、metadata_system_fields、metadata_field_type_mapping
- 其他表：依赖应用层雪花算法 (UidGenerator) 或 MAX+1

#### 1.3 CHECK约束转换
```sql
-- PostgreSQL: CHECK ((data_type)::text = ANY (ARRAY[...]))
-- 达梦: CHECK ("data_type" IN ('BOOLEAN', 'INTEGER', ...))
```

#### 1.4 Schema处理
- 移除 schema 前缀（public.），直接使用表名
- 所有表名、列名加双引号保持大小写敏感
- 索引定义统一加双引号（204处索引，约1200个列）

#### 1.5 执行脚本
```bash
disql SYSDBA/密码@localhost:5236 \`sql/dameng/onebase_cloud_v3_dm.sql
```

### 2. 数据初始化脚本 DML

#### 2.1 关键配置
- 脚本位置：`sql/dameng/system_data_init_dm.sql`
- 文件头部必须包含：
  ```sql
  SET DEFINE OFF;  -- 禁用替换变量功能，解决&符号问题
  BEGIN;
  -- INSERT语句 (2310条)
  COMMIT;
  /
  ```

#### 2.2 执行方式
```bash
# 方式1：disql命令行（推荐）
disql SYSDBA/密码@localhost:5236 \`sql/dameng/system_data_init_dm.sql

# 方式2：disql交互模式
disql SYSDBA/密码@localhost:5236
SQL> SET DEFINE OFF;
SQL> START /path/to/system_data_init_dm.sql;
```

#### 2.3 常见问题
1. **最后一条SQL未执行**：文件末尾缺少 `COMMIT;` 和 `/` 以及换行符
2. **主键冲突**：表中已有数据，需先清空

#### 2.4 验证
```sql
SELECT COUNT(*) FROM "system_users";
SELECT * FROM "onebase_demo03_student" WHERE "id" = 9;
SELECT "name" FROM "system_menu" WHERE "name" LIKE '%退款%';  -- 应返回：支付&退款案例
```

### 4. 应用代码改造

#### 4.1 WorkerNodeDAO.addWorkerNode
- 位置：`WorkerNodeDAO.java` (line 83-102)
- 改动：达梦采用手动 MAX+1 方式生成ID
  ```java
  Long maxId = jdbcTemplate.queryForObject(
      "SELECT COALESCE(MAX(\"id\"), 0) FROM \"system_uid_worker_node\"", Long.class);
  Long newId = (maxId != null ? maxId : 0L) + 1;
  ```
- 目的：达梦 system_uid_worker_node 表无自增列
- **注意**：高并发场景可能存在竞态条件，建议使用分布式锁或达梦序列

#### 4.2 数据库类型检测
- 位置：`WorkerNodeDAO.getDatabaseType()` (line 50-77)
- 检测逻辑：driver name 包含 "dm" 或 "dameng"

#### 4.3 连接测试查询

## 部署顺序

### KingBase
1. 创建数据库（兼容模式PG）
2. 执行 DDL：`sql/kingbase/onebase_cloud_v3_kingbase.sql`
3. 执行 DML（如存在）
4. 同步序列值
5. 启动应用测试

### Dameng
1. 创建数据库（UTF-8编码）
2. 执行 DDL：`sql/dameng/onebase_cloud_v3_dm.sql`
3. 执行 DML：`sql/dameng/system_data_init_dm.sql`
4. 执行 Patch（可选）
5. 验证数据完整性
6. 启动应用测试
