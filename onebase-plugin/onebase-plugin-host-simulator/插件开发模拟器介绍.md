# OneBase 插件开发模拟器 (Plugin Host Simulator)

本工具专为 OneBase 插件开发者设计，支持独立的插件开发、调试和验证。

## 🚀 三种标准模式 (Standard Modes)

我们推荐采用以下三种标准模式，覆盖从开发到发布的完整流程。

### 1. 开发模式 (Development)
**场景**：日常业务开发、逻辑验证、快速调试。
- **核心方式**：**本地独立运行**。
- **操作步骤**：
  1. 在 IDE 中直接运行插件工程下的启动类（如 `OcrPluginApplication`）。
  2. 通过修改 `src/test/resources/application.yml` 中的 `spring.profiles.active` 切换 `dev` 或 `prod` 配置。
- **优势**：
  - **秒级启动**：无需加载宿主环境，开发效率极高。
  - **原生调试**：直接利用 IDE 的断点调试和热部署功能。
  - **配置灵活**：通过本地 `application-dev.yml` 和 `application-prod.yml` 快速验证不同环境的配置逻辑。

### 2. 模拟生产模式 (Simulated Production)
**场景**：验证插件与宿主的集成、ClassLoader 隔离性、ZIP 包完整性。
- **核心方式**：**模拟器集成运行**。
- **前置条件**：需先执行 Maven 打包生成 ZIP 文件。
- **操作步骤**：
  1. 修改模拟器启动脚本 (`start-dev.sh/bat`)，设置参数：
     ```bash
     --onebase.plugin.mode=staging
     --onebase.plugin.plugins-dir=/path/to/your/plugins-dist
     ```
  2. 启动模拟器。
- **优势**：
  - **环境仿真**：在完全隔离的 ClassLoader 中运行，提前发现类冲突问题。
  - **全生命周期**：验证插件的加载、启动、停止、卸载等完整流程。

### 3. 直接生产打包 (Direct Production Packaging)
**场景**：发布插件、交付产物。
- **核心方式**：**Maven 构建**。
- **操作步骤**：
  在插件工程根目录下运行：
  ```bash
  mvn clean package
  ```
- **产物说明**：
  构建完成后，在 `target` 目录下生成的 `plugin-xxx.zip` 即为最终交付物，可直接在 OneBase 生产环境安装使用。

## 🛠️ 常用命令与配置速查

### 模拟器启动参数 (仅模拟生产模式需要)

| 参数项 | 说明 | 示例 |
| :--- |:---|:---|
| `onebase.plugin.mode` | 运行模式 (固定为 `staging`) | `staging` |
| `onebase.plugin.plugins-dir` | 插件 ZIP 包存放目录 (绝对路径) | `/Users/dev/plugins` |

### IntelliJ IDEA 调试指南

- **开发模式**：直接点击 `Run/Debug` 按钮启动 `Application` 类。
- **模拟生产模式**：
  1. 启动模拟器脚本。
  2. 在 IDE 中创建 **Remote JVM Debug** 配置 (端口 5005)。
  3. 点击 Debug 连接模拟器。
