#### 2. 自动配置改造

- **改造内容**：

  - STAGING 模式：根据 `autoLoad` 和 `autoStart` 控制插件加载和启动

  - 所有模式均支持配置灵活控制，增强启动阶段的可控性

#### 3. 生命周期事件体系
- **新增事件类**（位于 `runtime.event` 包）：
  - `PluginBeforeLoadEvent`：插件加载前事件
  - `PluginBeforeStartEvent`：插件启动前事件
  - `PluginBeforeStopEvent`：插件停止前事件
  - `PluginBeforeUnloadEvent`：插件卸载前事件
  - `PluginLoadFailedEvent`：插件加载失败事件
  - `PluginStartFailedEvent`：插件启动失败事件
- **事件特性**：
  - 携带时间戳、操作者信息
  - 支持监听器订阅和处理
  - 为后续审计和监控提供基础

#### 4. 插件管理器增强
- **文件**：`OneBasePluginManager.java`
- **新增方法**：
  - `loadPluginWithRollback(Path, String)`：带回滚机制的插件加载
  - `startPluginWithRollback(String, String)`：带回滚机制的插件启动
  - `stopPluginWithEvent(String, String)`：带事件通知的插件停止
  - `unloadPluginWithEvent(String, String)`：带事件通知的插件卸载
- **核心特性**：
  - 完整的生命周期事件发布（前置事件 + 后置事件 + 失败事件）
  - 加载失败时自动回滚（卸载已部分加载的插件）
  - 支持操作者追踪，便于审计
  - 保留原有方法向后兼容
