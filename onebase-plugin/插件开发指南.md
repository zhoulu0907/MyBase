# OneBase 插件开发指南

本文档旨在为插件开发者提供一份详尽的指南，帮助您快速上手开发 OneBase 插件，涵盖从环境搭建、代码编写、调试热重载到打包发布的全生命周期流程。

## 1. 简介

OneBase 插件系统基于 **PF4J** 框架构建，旨在通过低代码平台扩展高代码能力。开发者可以通过 Java 插件的形式，扩展系统的 API 接口、监听业务事件或实现复杂的数据处理逻辑，同时享受平台提供的上下文隔离和生命周期管理能力。

### 核心概念

- **扩展点 (Extension Point)**: 平台定义的接口契约，插件通过实现这些接口来注入功能。主要包括：
    - `HttpHandler`: 定义 REST API 接口。
    - `EventListener`: 监听平台业务事件（如数据创建、用户登录）。
    - `DataProcessor`: 数据导入/导出及转换处理。
- **插件上下文 (PluginContext)**: 插件运行时的上下文环境，用于获取平台核心服务（Service）和 Bean。
- **运行模式**:
    - **DEV (开发模式)**: 从 class 目录加载，具备**热重载**能力，专为开发调试设计。
    - **STAGING (预发布模式)**: 从 ZIP 包加载，模拟生产环境加载策略，用于验证打包和全生命周期。
    - **PROD (生产模式)**: 从 ZIP 包加载，生产环境的标准运行模式。

## 2. 快速开始 (Quick Start)

### 2.1 环境准备
- **JDK**: 17+
- **Maven**: 3.8+
- **IDE**: IntelliJ IDEA (推荐)

### 2.2 创建插件项目

推荐创建一个标准的 Maven 项目。

**1. 配置 pom.xml**

在 `pom.xml` 中引入 `onebase-plugin-sdk` 依赖，并配置构建插件。

```xml
<dependencies>
    <!-- 引入插件 SDK -->
    <dependency>
        <groupId>com.cmsr.onebase</groupId>
        <artifactId>onebase-plugin-sdk</artifactId>
        <version>${onebase.version}</version>
        <scope>provided</scope> <!-- 必须设为 provided，由平台提供 -->
    </dependency>
</dependencies>

<build>
    <plugins>
        <!-- OneBase 插件构建工具 -->
        <plugin>
            <groupId>com.cmsr.onebase</groupId>
            <artifactId>onebase-plugin-maven-plugin</artifactId>
            <version>${onebase.version}</version>
            <executions>
                <execution>
                    <id>package-plugin</id>
                    <goals>
                        <goal>generate-plugin-class</goal>
                        <goal>generate-properties</goal>
                        <goal>scan-extensions</goal>
                        <goal>package-plugin</goal>
                    </goals>
                </execution>
            </executions>
            <configuration>
                <pluginId>my-first-plugin</pluginId>
                <pluginVersion>1.0.0</pluginVersion>
                <pluginDescription>My first OneBase plugin</pluginDescription>
                <pluginProvider>My Company</pluginProvider>
            </configuration>
        </plugin>
    </plugins>
</build>
```

### 2.3 编写第一个插件

创建一个 `HttpHandler` 来处理 HTTP 请求。

```java
package com.example.plugin;

import com.cmsr.onebase.plugin.api.HttpHandler;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;
import java.util.Map;

@RestController
@RequestMapping("/plugin/my-first-plugin") // ❗ 必须遵循 /plugin/{pluginId}/ 规范，否则maven编译打包时会失败，运行期也会报异常！
public class HelloController implements HttpHandler {

    @GetMapping("/hello")
    public Map<String, String> hello() {
        return Map.of("message", "Hello from OneBase Plugin!");
    }
}
```

### 2.4 运行验证

我们使用 **OneBase 插件模拟器 (Plugin Host Simulator)** 来运行插件。

1.  **启动模拟器**: 使用提供的启动脚本 `start-dev.bat` (Windows) 或 `start-dev.sh` (Mac/Linux)。
    需要配置 `--onebase.plugin.dev-class-paths` 指向你插件的编译输出目录。
    ```bash
    # 例如 Windows 下
    start-dev.bat --onebase.plugin.dev-class-paths="D:/work/my-first-plugin/target/classes"
    ```
2.  **测试接口**: 访问 `http://localhost:8080/plugin/my-first-plugin/hello`。

## 3. 核心开发指南

### 3.1 扩展点详解

OneBase 提供了多种扩展点以满足不同业务需求。

#### 3.1.1 HTTP 接口开发 (HttpHandler)
- **用途**: 对外暴露 RESTful API。
- **实现**: 实现 `HttpHandler` 接口（标记接口），并使用标准的 Spring MVC 注解 (`@RestController`, `@RequestMapping`)。
- **规范**: 路由必须以 `/plugin/{pluginId}/` 开头，平台会进行校验和拦截。

#### 3.1.2 业务事件监听 (EventListener)
- **用途**: 监听平台发生的业务动作，如数据变更、流程流转、用户登录等。
- **接口的方法**:
    - `eventTypes()`: 返回需要监听的事件类型数组 (如 `DATA_CREATED`, `USER_LOGIN`)。
    - `onEvent(PluginContext ctx, PluginEvent event)`: 事件处理逻辑。
- **示例**:
    ```java
    import com.cmsr.onebase.plugin.api.EventListener;
    
    public class DataAuditListener implements EventListener {
        @Override
        public String[] eventTypes() {
            return new String[]{"DATA_CREATED", "DATA_UPDATED"};
        }

        @Override
        public void onEvent(PluginContext ctx, PluginEvent event) {
            System.out.println("收到数据变更事件: " + event.getData());
        }
    }
    ```

#### 3.1.3 数据处理器 (DataProcessor)
- **用途**: 参与数据的导入、导出或复杂转换流程。
- **接口方法**:
    - `type()`: 处理器唯一标识。
    - `process(PluginContext ctx, Object input, Map config)`: 执行处理逻辑。

### 3.2 使用平台服务 (PluginContext)

插件不应直接依赖平台具体实现类，而应通过 `PluginContext` 获取相应服务接口。

```java
public void doSomething(PluginContext context) {
    // 获取用户服务
    UserService userService = context.getService(UserService.class);
    // 获取当前登录用户
    User currentUser = userService.getCurrentUser();
}
```

## 4. 调试与热重载 (Debugging & Hot Reload)

### 4.1 开发模式 (DEV Mode)
- **配置**: `onebase.plugin.mode=dev`
- **原理**: 模拟器直接监控 `dev-class-paths` 指定的目录（通常是 `target/classes`），监听 `.class` 文件变更。

### 4.2 热重载工作流
1. **启动模拟器**: 配置正确的 `dev-class-paths` 并启动。
2. **修改代码**: 在 IDE 中修改 Java 源码。
3. **编译 (Compile/Build)**: 在 IDE 中执行编译（Ctrl+F9 / Cmd+F9）。**注意：必须编译生成新的 .class 文件，热重载才会触发。**
4. **自动生效**: 模拟器控制台会提示 "触发热重载"，看到"热重载成功"，可直接调试代码无需重启 JVM。

### 4.3 远程调试 (Remote Debug)
模拟器默认在 **5005** 端口开启 JDWP 调试监听。

1. **IDE 配置**: 在 IntelliJ IDEA 中创建 "Remote JVM Debug" 配置。
    - Host: `localhost`
    - Port: `5005`
    - **Debugger mode**: 必须选择 **Attach to remote JVM** (因为模拟器作为 Server 监听端口)。
    - **Use module classpath**: 选择你的**插件模块** (关键！否则断点无效)。
2. **启动调试**: 
    - **第一步**: 启动模拟器 (`start-dev.bat` 或 `start-dev.sh`)。
    - **第二步**: 待模拟器启动后，在 IDE 点击 Debug 按钮连接。
3. **断点**: 在源码中打断点，发起请求即可触发。

## 5. 打包与生命周期验证

开发完成后，建议在 Staging 模式下验证插件的完整生命周期。

### 5.1 构建插件包
在项目根目录执行：
```bash
mvn clean package
```
构建成功后，`target` 目录下会生成一个 ZIP 文件（如 `my-plugin-1.0.0.zip`）。
**ZIP 包结构**:
- `classes/`: 插件编译后的 class 文件
- `lib/`: 插件依赖的第三方 jar 包 (不含 SDK)
- `plugin.properties`: 插件描述文件

### 5.2 预发布验证 (STAGING Mode)
Staging 模式完全模拟生产环境加载逻辑（加载 ZIP 包，不扫描 classpath）。

1. **准备**: 将打好的 ZIP 包放入一个目录（例如 `D:/plugin-dist`）。
2. **配置启动**:
    ```bash
    start-dev.bat --onebase.plugin.mode=staging --onebase.plugin.plugins-dir="D:/plugin-dist"
    ```
3. **验证**:
    - 观察控制台日志，确认插件被正确加载 (Load) 和启动 (Start)。
    - 测试功能是否正常。
    - 可以尝试在运行时删除/添加 ZIP 包，验证动态热插拔能力。

## 6. 最佳实践与常见问题 (FAQ)

### 6.1 最佳实践
- **依赖管理**: `onebase-plugin-sdk` 必须配合 `<scope>provided</scope>` 使用，避免打入插件包导致版本冲突。
- **第三方库**: 插件独有的第三方库正常引入即可，Maven 插件会自动将其打包进 ZIP 的 `lib` 目录，并由插件独立的 ClassLoader 加载，不会与主程序或其他插件冲突。
- **路径规范**: 严格遵守 `/plugin/{pluginId}/` 路由前缀，否则会被拦截器拒绝。

### 6.2 常见问题

**Q: 修改代码后热重载没反应？**
A: 请确认你是否执行了 IDE 的 **Compile/Build** 操作。模拟器监听的是磁盘上的 `.class` 文件，单纯保存 `.java` 源码是无效的。

**Q: 远程调试断点打不住？**
A: 请检查 Remote Debug 配置中的 "Use module classpath" 是否选择了你的插件模块。如果选了 `<no module>` 或者模拟器模块，IDE 无法映射源码。

**Q: 开发模式下能加载 ZIP 包吗？**
A: 不能。DEV 模式仅加载 `dev-class-paths` 中的 class 文件。如需测试 ZIP 包，请切换到 STAGING 模式。

**Q: 多个插件可以使用相同的 RequestMapping 路径吗？**
A: 不可以。虽然它是微服务架构下的插件，但在同一个 Spring 上下文中，路径冲突会导致加载失败。请务必使用 pluginId 作为路径前缀进行隔离。
