# Starter 目标
  - 支持三种运行模式（DEV/ STAGING/ PROD），满足不同环境下插件开发、测试和生产的需求；
  - 统一插件加载、启动、停止、卸载等生命周期管理能力，使 starter 同时支持插件的静态（启动时一次性）和动态（运行时加载/启动/停止/卸载等）全生命周期管理能力；
  - 将 starter 中职责雷同的类合并为高内聚、低耦合的模块（先聚焦 HTTP 处理），并建立逐步合并非 HTTP 模块的规范流程。
# 目标详情: 
## 三种运行模式
### 1. DEV模式（开发模式）
- **配置**: `onebase.plugin.mode=dev`
- **加载策略**: 只加载classpath下的扩展点
- **适用场景**:
    - IDE中直接启动和调试
    - 支持断点调试
    - 无需打包ZIP/JAR
    - 快速开发迭代
- **特点**:
    - IDE中直接启动
    - 断点调试：完整的 IDE 调试支持
    - 无需打包插件ZIP/JAR
    - 仅扫描classpath

### 2. STAGING模式（预发布模式/全生命周期验证模式）
- **配置**: `onebase.plugin.mode=staging`
- **加载策略**: 只加载plugin目录的ZIP/JAR包，不扫描classpath
- **适用场景**:
    - 验证插件完整生命周期（加载、启动、热插拔、卸载）
    - 测试ClassLoader隔离
    - 验证ZIP包完整性
    - 模拟真实生产环境
- **特点**:
    - 需要打包插件ZIP/JAR
    - 仅扫描pugin目录

### 3. PROD模式（生产模式）
- **配置**: `onebase.plugin.mode=prod`（默认值）
- **加载策略**: 使用PF4J默认策略，同时扫描classpath和plugin目录
- **适用场景**:
    - 生产环境
    - 提供最大灵活性
- **特点**:
    - 需要打包插件ZIP/JAR
    - classpath + plugin目录都扫描
    - 灵活但不同加载器可能重复加载扩展点
  - 保严格校验 `onebase.plugin.mode` 的合法值（dev/staging/prod）；
  - 避免在 Spring 自动配置的类加载阶段触发插件类的类加载。

## 插件全生命周期管理
### 1. 插件静态加载能力
  - simulator-host或宿主应用启动时根据配置参数一次性加载、启动所有插件。比如：`onebase.plugin.enable=true`、`onebase.plugin.auto-load=true`、`onebase.plugin.auto-start=true`时自动加载并启动所有插件，配置参数用户可自自由配置。
### 2. 插件动态管理能力
  - 在宿主应用运行过程中，支持动态加载、启动、停止、卸载插件。比如：通过管理控制台上传插件ZIP包并动态加载、启动插件，或者通过管理控制台停止、卸载插件等操作。
### 3. 生命周期事件通知
  - 在插件生命周期的各个阶段（加载前、加载后、启动前、启动后、停止前、停止后、卸载前、卸载后等）提供事件通知机制，允许插件或宿主应用监听和响应这些事件。
### 4. 错误处理和回滚机制
  - 在插件生命周期管理过程中，提供错误处理和回滚机制，确保在插件加载、启动、停止、卸载过程中出现错误时能够安全地回滚到之前的状态，保证系统的稳定性。
## 统一模块规范流程
### 1. 模块职责划分
  - 明确每个模块的职责范围，确保模块之间的高内聚和低耦合。比如：HTTP处理模块专注于处理HTTP请求和响应，插件管理模块专注于插件的加载和生命周期管理等。
### 2. 代码合并规范
  - 制定代码合并的规范流程，确保在合并职责雷同的类时，遵循模块化设计原则，保持代码的可维护性和可读性。
### 3. 测试覆盖
  - 在合并模块时，确保有充分的测试覆盖，包括单元测试、集成测试和端到端测试，确保模块功能的正确性和稳定性。
### 4. 文档更新
  - 在模块合并和职责划分过程中，及时更新相关的技术文档，确保团队成员能够了解最新的模块设计和职责划分
